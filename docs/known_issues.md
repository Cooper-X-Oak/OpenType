# 已知问题记录 (Known Issues Log)

## 1. 热键捕获 (Hotkey Capture)

### 1.1 修饰键立即触发 (Standalone Modifier Trigger)
- **现象**: 在自定义热键界面，按下单独的修饰键（如 Ctrl, Shift, Alt）时，系统会立即识别为 `ctrl`, `shift` 等并更新显示，而不是等待用户按下组合键（如 `ctrl+c`）。
- **影响**: 用户难以输入组合键，往往刚按下 Ctrl 就被判定为选择了 Ctrl 键。虽然用户可以通过按住不放再按其他键来尝试覆盖，但体验不佳。
- **期望**: 应该在修饰键按下时不立即确认，或者等待按键释放后再确认（但这会影响组合键的实时显示）。理想交互是：按下修饰键显示 "Ctrl + ...", 等待第二个键。

### 1.2 输入框冲突 (Input Box Conflict) - 已绕过
- **现象**: 之前使用隐藏的 `QLineEdit` 进行捕获，导致无法识别单独的修饰键，且回车键会被输入框吞掉。
- **状态**: 已通过移除输入框、改用 `keyPressEvent` 全局捕获解决。

### 1.3 视觉反馈 (Visual Feedback)
- **现象**: 某些特殊按键或在不同系统环境下，`QKeySequence` 可能无法返回正确的文本，导致显示为空。
- **状态**: 已添加 `event.text()` 作为兜底，但复杂的组合键显示可能仍有边缘情况（如 `Win` 键）。

## 2. 下一步计划 (Next Steps)
- 在新的 Change 中专门优化热键捕获交互（参考 VSCode 或其他 IDE 的按键设置体验）。
- 考虑引入 `keyboard` 库进行更底层的钩子捕获（如果 Qt 事件机制限制太多）。
